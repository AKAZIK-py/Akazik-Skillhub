## 二、怎么选？决策树

```
你的评测目标是什么？
│
├─► "快速对比两个模型的整体水平"
│   └─► 选 DrawBench（经典200条，全维度覆盖）
│       或 GenEval（553条，有结构化标注便于自动评分）
│
├─► "深度测试组合生成能力（属性绑定、空间关系）"
│   └─► 选 T2I-CompBench（专攻组合，自动化评估指标最完备）
│
├─► "压力测试长prompt/复杂指令遵循"
│   └─► 选 DPG-Bench（平均67词/prompt，极限挑战）
│
├─► "需要自动化评分pipeline"
│   └─► 选 TIFA（自带VQA验证问题，脚本直接跑）
│       或 T2I-CompBench（自带BLIP-VQA/UniDet评估代码）
│
├─► "需要全面覆盖、不遗漏维度"
│   └─► 选 HRS-Bench（13个维度最全）
│       或 PartiPrompts（1632条，难度分级最细）
│
├─► "重点关注中文能力"
│   └─► 开源选择有限，建议 FlagEval + 自建
│
└─► "本项目（nanobanana2 vs Seedream）"
    └─► 推荐组合：DrawBench + T2I-CompBench + GenEval
        + 自建中文补充集（见下文）
```

### 2.1 本项目推荐选择（带理由）

| 优先级 | 评测集 | 取多少条 | 为什么选 |
|--------|--------|---------|----------|
| 🥇 核心 | **T2I-CompBench** | ~50条 | 组合生成是当前模型差异最大的维度，自动化评估最成熟 |
| 🥈 骨架 | **DrawBench** | ~40条 | 业界标准baseline，论文可引用，维度全面 |
| 🥉 补充 | **GenEval** | ~30条 | 结构化标注，方便清单制打分 |
| 🏅 自建 | **中文场景补充** | ~30条 | 开源集不覆盖中文，必须自建 |
| **合计** | | **~150条** | |

### 3.3 第三步：分层筛选策略

```
筛选原则：
1. 能力维度全覆盖（12个维度每个都要有代表）
2. 难度梯度均衡（简单:中等:困难 = 3:4:3）
3. 去重去相似（同维度的prompt不能太雷同）
4. 可评判性（prompt要有明确对错标准，不能太主观）
```

#### 从T2I-CompBench筛50条

```python
"""
T2I-CompBench筛选策略：6个维度各取8-9条
"""
import pandas as pd
import random

# 每个维度的筛选规则
selection_rules = {
    "color_binding": {
        "count": 9,
        "strategy": "覆盖：单物体单色/双物体双色/三物体三色，难度递进"
    },
    "shape_binding": {
        "count": 8,
        "strategy": "覆盖：基本形状/复合形状/抽象形状描述"
    },
    "texture_binding": {
        "count": 8,
        "strategy": "覆盖：常见材质/罕见材质/对比材质"
    },
    "spatial": {
        "count": 9,
        "strategy": "覆盖：上下/左右/前后/里外/远近/嵌套空间"
    },
    "non_spatial": {
        "count": 8,
        "strategy": "覆盖：动作关系/使用关系/情感关系"
    },
    "complex": {
        "count": 8,
        "strategy": "覆盖：2属性叠加/3属性叠加/4属性叠加"
    }
}

# 筛选流程（伪代码）
for dimension, rule in selection_rules.items():
    pool = load_prompts(dimension)  # 每个维度1000条
    
    # Step 1: 按复杂度排序（物体数量、属性数量、关系数量）
    pool = sort_by_complexity(pool)
    
    # Step 2: 分三档取样（简单/中等/困难）
    easy = pool[:333]
    medium = pool[333:666]
    hard = pool[666:]
    
    n = rule["count"]
    selected = (
        sample(easy, n // 3) +      # 简单档
        sample(medium, n // 3 + 1) + # 中等档（多1条）
        sample(hard, n // 3)         # 困难档
    )
    
    # Step 3: 去相似（用sentence-embedding计算相似度，>0.85的去重）
    selected = deduplicate(selected, threshold=0.85)
```

### 3.4 第四步：汇总筛选脚本（可执行）

```python
"""
完整筛选流程脚本
输入：各开源评测集原始数据
输出：final_150_prompts.csv
"""
import pandas as pd
import json
from sentence_transformers import SentenceTransformer
from sklearn.metrics.pairwise import cosine_similarity
import numpy as np

# ============ 配置 ============
OUTPUT_FILE = "final_150_prompts.csv"
SIMILARITY_THRESHOLD = 0.85  # 相似度高于此值的去重

# ============ 工具函数 ============
def load_drawbench(path="drawbench_prompts.csv"):
    """加载DrawBench，返回带category标签的DataFrame"""
    df = pd.read_csv(path)
    return df  # columns: [prompt, category]

def load_t2i_compbench(path="t2i_compbench/"):
    """加载T2I-CompBench各维度"""
    dimensions = {}
    for dim in ["color", "shape", "texture", "spatial", "non_spatial", "complex"]:
        with open(f"{path}/{dim}.txt") as f:
            dimensions[dim] = [line.strip() for line in f.readlines()]
    return dimensions

def load_geneval(path="geneval/prompts/"):
    """加载GenEval，返回带结构化标注的列表"""
    prompts = []
    with open(f"{path}/evaluation_metadata.jsonl") as f:
        for line in f:
            prompts.append(json.loads(line))
    return prompts

def deduplicate(prompts, model, threshold=0.85):
    """基于语义相似度去重"""
    embeddings = model.encode(prompts)
    sim_matrix = cosine_similarity(embeddings)
    
    keep = []
    removed = set()
    for i in range(len(prompts)):
        if i in removed:
            continue
        keep.append(i)
        for j in range(i+1, len(prompts)):
            if sim_matrix[i][j] > threshold:
                removed.add(j)
    
    return [prompts[i] for i in keep]

def stratified_sample(prompts, n, complexity_fn):
    """分层抽样：按复杂度分三档"""
    scored = [(p, complexity_fn(p)) for p in prompts]
    scored.sort(key=lambda x: x[1])
    
    third = len(scored) // 3
    easy = [p for p, _ in scored[:third]]
    medium = [p for p, _ in scored[third:2*third]]
    hard = [p for p, _ in scored[2*third:]]
    
    n_easy = n // 3
    n_hard = n // 3
    n_medium = n - n_easy - n_hard
    
    import random
    result = (
        random.sample(easy, min(n_easy, len(easy))) +
        random.sample(medium, min(n_medium, len(medium))) +
        random.sample(hard, min(n_hard, len(hard)))
    )
    return result

def prompt_complexity(prompt):
    """简单的复杂度估计：词数 + 逗号数 + and/with数量"""
    words = prompt.split()
    commas = prompt.count(",")
    conjunctions = prompt.lower().count(" and ") + prompt.lower().count(" with ")
    return len(words) + commas * 2 + conjunctions * 3

# ============ 主流程 ============
def build_evaluation_set():
    model = SentenceTransformer('all-MiniLM-L6-v2')  # 用于去重
    
    all_selected = []
    
    # --- 1. T2I-CompBench: 50条 ---
    t2i = load_t2i_compbench()
    t2i_counts = {"color": 9, "shape": 8, "texture": 8, 
                  "spatial": 9, "non_spatial": 8, "complex": 8}
    
    for dim, n in t2i_counts.items():
        sampled = stratified_sample(t2i[dim], n, prompt_complexity)
        for p in sampled:
            all_selected.append({
                "prompt": p,
                "source": "T2I-CompBench",
                "dimension": dim,
                "language": "en"
            })
    
    # --- 2. DrawBench: 40条 ---
    db = load_drawbench()
    db_counts = {
        "Positional": 6, "Counting": 5, "Colors": 4, "Text": 5,
        "Conflicting": 4, "Descriptions": 4, "DALL-E": 3, "Reddit": 4,
        "Gary Marcus": 2, "Rare Words": 2, "Misspellings": 1
    }
    
    for cat, n in db_counts.items():
        pool = db[db["category"] == cat]["prompt"].tolist()
        sampled = stratified_sample(pool, n, prompt_complexity)
        for p in sampled:
            all_selected.append({
                "prompt": p,
                "source": "DrawBench",
                "dimension": cat,
                "language": "en"
            })
    
    # --- 3. GenEval: 30条 ---
    ge = load_geneval()
    # 按子类别筛选...
    # (类似逻辑)
    
    # --- 4. 中文自建: 30条 ---
    chinese_prompts = [
        # 成语典故
        {"prompt": "画蛇添足：一个人在已经画好的蛇旁边画脚", "dimension": "idiom"},
        {"prompt": "守株待兔：一个农夫坐在树桩旁等待", "dimension": "idiom"},
        # ... 补齐30条
    ]
    for p in chinese_prompts:
        all_selected.append({
            "prompt": p["prompt"],
            "source": "custom_chinese",
            "dimension": p["dimension"],
            "language": "zh"
        })
    
    # --- 5. 全局去重 ---
    en_prompts = [x["prompt"] for x in all_selected if x["language"] == "en"]
    en_prompts_deduped = deduplicate(en_prompts, model, SIMILARITY_THRESHOLD)
    
    # 过滤掉被去重的
    en_set = set(en_prompts_deduped)
    all_selected = [x for x in all_selected 
                    if x["language"] == "zh" or x["prompt"] in en_set]
    
    # --- 6. 输出 ---
    df = pd.DataFrame(all_selected)
    df.index.name = "prompt_id"
    df.to_csv(OUTPUT_FILE, index=True)
    
    print(f"最终评测集：{len(df)}条")
    print(f"来源分布：\n{df['source'].value_counts()}")
    print(f"维度分布：\n{df['dimension'].value_counts()}")
    
    return df

if __name__ == "__main__":
    build_evaluation_set()
```

### 4.3 自动化评分方案（利用开源集标注）

#### 方案A：基于GenEval标注 + 目标检测

```python
"""
利用GenEval的结构化标注实现半自动评分
"""
from transformers import Owlv2ForObjectDetection, Owlv2Processor
from PIL import Image

def auto_eval_geneval(image_path, expected_objects):
    """
    用OWLv2检测生成图片中是否包含期望物体
    
    expected_objects: [
        {"class": "bird", "count": 1, "color": "blue"},
        {"class": "car", "count": 1, "color": "red"}
    ]
    """
    model = Owlv2ForObjectDetection.from_pretrained("google/owlv2-base-patch16")
    processor = Owlv2Processor.from_pretrained("google/owlv2-base-patch16")
    
    image = Image.open(image_path)
    
    results = {}
    for obj in expected_objects:
        # 检测物体是否存在
        texts = [f"a {obj['class']}"]
        inputs = processor(text=texts, images=image, return_tensors="pt")
        outputs = model(**inputs)
        
        # 计算检测到的数量
        scores = outputs.logits.sigmoid().max(dim=-1).values
        detected_count = (scores > 0.3).sum().item()
        
        results[obj["class"]] = {
            "expected_count": obj["count"],
            "detected_count": detected_count,
            "match": detected_count == obj["count"]
        }
    
    # 总遵循率
    total = len(expected_objects)
    matched = sum(1 for r in results.values() if r["match"])
    compliance_rate = matched / total
    
    return results, compliance_rate
```

### 4.4 三种方案的对比

## 六、避坑指南

| 坑 | 怎么避 |
|----|--------|
| 开源集prompt太简单，区分度不够 | 用DPG-Bench补充10-20条高难度长prompt |
| 英文prompt翻译成中文后语义偏移 | 不翻译！英文prompt直接用英文测，中文另建补充集 |
| 不同开源集的prompt风格差异大 | 没关系，真实用户prompt本来就风格各异 |
| 自动化评分和人工评分不一致 | 以人工为准，自动化只做初筛和效率工具 |
| 样本量太少统计不显著 | 每条prompt生成3次可以提升样本量，或追加重点维度prompt |
| 评测集被模型"刷过"（数据泄漏） | 修改10-20%的prompt措辞（paraphrase），检验一致性 |

> ⚠️ **关于数据泄漏的特别提醒：**
> DrawBench和PartiPrompts太有名了，新模型训练时可能有意优化过这些case。
> 对策：
> 1. 对20%的经典prompt做改写（保持语义不变但措辞不同）
> 2. 对比原版和改写版的得分，如果差异大 → 可能存在过拟合
> 3. 自建的30条中文prompt天然没有泄漏风险
